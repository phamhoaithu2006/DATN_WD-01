# Test Case — Booking, định giá và VNPAY (BR-021–BR-034)

## Quy ước

- Các case dưới đây là test design; các case hậu sửa BUG-AB-001–004 được cập nhật theo source/test tự động hiện tại.
- Trường `Hiện trạng tự động hóa` dựa trên nội dung setup và assertion thực tế trong test hiện hữu, không dựa riêng vào tên test.
- Với callback VNPAY, payload “ký hợp lệ” nghĩa là hash HMAC SHA-512 được tạo sau khi sắp xếp/encode đúng như `VnpayService`.
- BUG ID lịch sử được giữ lại và gắn `Resolved`; kết quả mong đợi trong bảng là hành vi hậu sửa.
